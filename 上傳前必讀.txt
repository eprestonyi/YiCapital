■ 後端已到 v8.1（三市場自動淨值＋六個基準＋完整內容管理＋PDF 拖放更換）：
  ⚠ 必須先部署 worker/worker.js v8.1，再上傳網站文件；舊 v6 不認得 HK/A
     發布路徑，若順序倒轉，後台選 HK/A 仍可能覆蓋 US 工作簿。
  1. Cloudflare → Workers → yicapital-portal → 用 worker/worker.js 全量替換舊代碼 → Deploy（/api/health 應顯示 v8.1）
  2. Worker → Settings → Triggers → Cron Triggers → 添加兩條：
       30 21 * * *   （美股收盤後，更新 US 淨值＋S&P 500/NASDAQ/DOW）
       0 9 * * *     （北京時間 17:00，更新 HK/A＋HSI/HSTECH/CSI300）
  3. 之後在後台「發布淨值表」依次選 US、HK、A，各發布一份工作簿。
     發布會從持倉、現金流、負債與資本記錄建立後台賬本；NAV Statement
     只保留歷史曲線。從次日起後台每天自算淨值、持倉權重和實時行，
     前台自動重算 Sharpe、最大回撤、VaR、壓測、Alpha/Beta 等。
  4. 以後只有交易、出入金、股息、公司行動或負債改變時才需重新發布；
     沒有交易時不需要為了補 NAV 每天上傳。

■ 後台已拆分為獨立頁面（yicapital.co/admin 進入總覽）：
  admin           總覽 + 系統狀態
  admin-publish   發布淨值表（發布即自動同步賬本）
  admin-reports   研報管理（自動載入全部；新增/修改/停用/刪除；PDF 拖放上傳/覆蓋）
  admin-insights  研究觀點管理（同上）
  admin-users     帳號管理（搜索用戶名/郵箱、停用、重置密碼、刪除）
  admin-mail      郵件中心（群發或指定用戶，帶搜索）
  admin-inbox     收件箱
  首次使用內容管理：進 admin-reports / admin-insights 後，系統會自動選入站內全部條目，
  再合併後端已有修改。若顯示「補回 N 條」，點一次「發布全部變更」即可保存完整合集。

■ PDF 上傳／更換（不需自己進 GitHub 拖 PDF）：
  1. 進 admin-reports，在列表找到報告並點「編輯」。
  2. 騰訊報告會自動帶出 assets/pdf/tencent-0700-ch12.pdf；
     把新版 PDF 拖入上傳框，即使本地檔名不同，也會覆蓋這個原文件名。
  3. 高階矩報告原本沒有 PDF；第一次拖入後會建立新文件並回填 PDF 路徑。
  4. 上傳成功後，依次點「保存此條」和「發布全部變更」。
  5. 繁／簡／EN 三個報告頁會讀取同一個後台 PDF 路徑，自動顯示下載及預覽。
